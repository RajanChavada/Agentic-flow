  <name>Task 1: Create TagInput reusable component</name>
  <files>frontend/src/components/ui/TagInput.tsx</files>
  <action>
Create a new file `frontend/src/components/ui/TagInput.tsx` with a reusable tag input component.

Props interface:
```typescript
interface TagInputProps {
  value: string[];
  onChange: (tags: string[]) => void;
  placeholder?: string;
  maxTags?: number;
}
```

Component requirements (per user decisions -- these are locked):

**Interactions:**
- Enter key: trim input, reject empty/whitespace, check for duplicate (case-insensitive). If duplicate, trigger pulse animation on matching chip. If unique, add to array, clear input.
- Backspace on empty input: remove last tag from array, put its text back into the input for editing.
- X button on chip: remove that tag from array.
- Anti-cascade protection: track Backspace via both `onKeyDown` and `onKeyUp` to prevent holding Backspace from rapidly deleting multiple tags. Use a ref `isBackspaceHeld` -- only act on Backspace if it was a fresh press (keyDown when ref is false), then set ref to true. Reset on keyUp.

**Visual styling (locked decisions):**
- 8-color palette cycling by index: `['sky-500', 'emerald-500', 'amber-500', 'rose-500', 'violet-500', 'cyan-500', 'orange-500', 'lime-500']`
- Chip classes: `bg-{color}/15 text-{color} border border-{color}/30 text-xs px-2 py-0.5 rounded-full inline-flex items-center gap-1`
- Dark mode chips: `dark:bg-{color}/20 dark:text-{color}-300`
  NOTE: Since Tailwind v4 won't JIT compile dynamic class strings, use a pre-built style map. Create a `CHIP_STYLES` array of 8 objects, each with explicit `bg`, `text`, `border`, `darkBg`, `darkText` class strings. Detect dark mode via `document.documentElement.classList.contains("dark")`.
- X button: Lucide `X` icon at `w-3.5 h-3.5`, `opacity-60 hover:opacity-100`, `cursor-pointer`
- Input: same visual weight as existing config modal inputs, `text-sm`, single-line, full-width below chips
- Container: wraps chips + input in a single field-like bordered container

**Animations:**
- Scale-in on add: apply a CSS class `animate-chip-in` that does `transform: scale(0) -> scale(1)` over 100ms ease-out. Define the keyframes via a `<style>` tag in the component or inline via a Tailwind `animate-[...]` utility if feasible.
- Pulse on duplicate: apply `animate-chip-pulse` class temporarily (300ms) to the matching chip. Use setTimeout to remove. Keyframes: brief scale(1.1) -> scale(1) with a subtle flash.

**Empty state:**
- When no tags exist and input is empty, show helper text below: "Define allowed actions (e.g., approve, reject, escalate)" styled as `text-xs text-zinc-400 dark:text-zinc-500 mt-1`

**Persistent label:**
- The label "Allowed Actions" is rendered by the PARENT (NodeConfigModal), not by TagInput. TagInput only renders the chips + input + helper text.

Mark with `"use client"` at top. Export as default.

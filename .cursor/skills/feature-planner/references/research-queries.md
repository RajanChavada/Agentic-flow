# Research Query Patterns

Use these query templates with Exa MCP (`web_search_exa`) during Step 4 of the feature planning process. Select 3-5 queries based on the feature category, plus 1-2 competitor queries.

---

## Query Templates by Feature Category

### Canvas / Drag-and-Drop UX
```
"best UX patterns for [feature type] in canvas-based tools like Figma Miro Lucidchart"
"drag and drop interaction design patterns web applications"
"canvas tool keyboard shortcuts and interaction patterns"
```

## Competitor Analysis Queries

Use 1-2 of these, customized to the feature domain:

```
"[competitor1] [competitor2] [feature type] implementation UX review"
"alternative to [product category] [feature type] comparison"
"[product type] feature tear-down UX analysis"
"best [feature type] implementations in [product category] tools"
```

### Common competitor sets by domain:

| Domain | Competitors to search |
|--------|----------------------|
| Canvas/workflow tools | Figma, Miro, Lucidchart, Whimsical, Excalidraw |
| AI/LLM tools | LangFlow, Flowise, Dify, Rivet, PromptFlow |
| Developer tools | Linear, Vercel, Railway, GitHub, GitLab |
| Analytics/dashboards | Mixpanel, Amplitude, PostHog, Datadog |
| SaaS general | Notion, Coda, Airtable, Retool |

## Research Extraction Prompt

When processing research results, extract:

1. **Actionable patterns** -- specific UX decisions you can replicate (not vague principles)
2. **Interaction details** -- transitions, animations, loading state approaches
3. **Information architecture** -- how content is organized, progressive disclosure patterns
4. **Error handling** -- how failures are communicated and recovered from
5. **Delight moments** -- unexpected pleasant details that create emotional connection
6. **Anti-patterns** -- common mistakes to avoid for this feature type
